_DTO hay tên gọi đầy đủ là Data Tranfer Object là một design pattern lần đầu tiên được giới thiệu bởi Martin Fowler trong cuốn sách EAA. 
Mục đích sử dụng chính của DTO đó là giảm số lần gọi các method giữa các tiến trình xử lý.

https://shareprogramming.net/dto-la-gi-dung-dto-trong-nhung-truong-hop-nao/

--------------------------------------------------------------------------------------------------------------------------------------
_ORM là viết tắt của "Object-Relational Mapping" trong lập trình. Đây là một kỹ thuật được sử dụng để ánh xạ dữ liệu giữa 
hệ quản trị cơ sở dữ liệu quan hệ (Relational Database Management System - RDBMS) và các đối tượng trong ngôn ngữ lập trình 
hướng đối tượng.


--------------------------------------------------------------------------------------------------------------------------------------
// Sử dụng object
object myObject = "Hello, world!";
int objectLength = ((string)myObject).Length; // Cần phải ép kiểu về string

// Sử dụng var
var myVar = "Hello, world!";
int varLength = myVar.Length; // Không cần phải ghi rõ kiểu dữ liệu

Tóm lại, object là một kiểu dữ liệu chung chung có thể chứa bất kỳ giá trị nào, trong khi var là một cách để khai báo biến 
mà không cần phải ghi rõ kiểu dữ liệu, với kiểu dữ liệu được xác định tại thời điểm biên dịch.
